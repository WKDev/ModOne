=== TASK F3b: RUBBER-BAND SEGMENT DRAG ===

CODE REVIEW FINDINGS
---------------------
File: src/components/OneCanvas/interaction/InteractionController.ts
Method: _startWireSegmentDragging() (lines 820-877)

Logic for connected port segments:
  Line 842: const isFromConnectedSegment = segHandleA === 0;
  Line 843: const isToConnectedSegment = segHandleA >= handleCount;

  Lines 845-852: When dragging the first segment (connected to 'from' port endpoint):
    if (isFromConnectedSegment && isPortEndpoint(wire.from)) {
      const fromPos = resolvePortEndpointPosition(wire.from, facade);
      if (fromPos) {
        facade.insertEndpointHandle(wireId, 'from', [
          { position: fromPos, constraint: 'free' },
        ]);
      }
    }

  Lines 854-861: When dragging the last segment (connected to 'to' port endpoint):
    if (isToConnectedSegment && isPortEndpoint(wire.to)) {
      const toPos = resolvePortEndpointPosition(wire.to, facade);
      if (toPos) {
        facade.insertEndpointHandle(wireId, 'to', [
          { position: toPos, constraint: 'free' },
        ]);
      }
    }

  Lines 866-876: After insertion, handle indices updated to point to the new free segment.

insertEndpointHandle is defined and wired in:
  - src/types/canvasFacade.ts:115 (interface)
  - src/stores/canvasStore.ts:1079 (implementation)
  - src/stores/hooks/useCanvasDocument.ts:561 (hook)
  - src/stores/hooks/useSchematicCanvasDocument.ts:592 (hook)
  - src/hooks/useCanvasFacade.ts:431 (facade bridge)

BEHAVIOR ANALYSIS:
  Before this change: Dragging a port-connected segment would directly move the port endpoint,
  breaking the port connection or causing invalid geometry.
  
  After this change: A new handle is inserted between the port position and the segment,
  creating a "rubber-band" bend. The port stays connected; the new intermediate handle
  moves freely during the drag.

VERDICT: Rubber-Band Drag PASS (code confirms correct implementation)
  - insertEndpointHandle called correctly for both from/to port connections ✅
  - Handle indices updated after insertion ✅
  - resolvePortEndpointPosition used to get exact port position ✅
  - Implementation fully wired through to canvasStore ✅
